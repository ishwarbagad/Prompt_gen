📘 HOW TO USE EMBEDGEN PRO
Complete step-by-step guide from start to finish
01
🔧 SELECT YOUR MCU
Go to the MCU tab. Search by name (e.g. "STM32F407") or browse by family filter. Click your microcontroller to select it. You can also add a custom MCU not in the list.
💡 The clock speed auto-fills from the MCU. Change it manually if you use a different PLL config.
02
⚙ CHOOSE FRAMEWORK & SETTINGS
Below the MCU grid, select your Framework (e.g. STM32 HAL, ESP-IDF, FreeRTOS+HAL), Language (C/C++), and Code Style. Add a project name and target application.
💡 For beginners use HAL. For production use LL (Low Layer) for better performance.
03
⚡ SELECT PERIPHERALS
Go to Peripherals tab. Click all MCU peripherals your project uses — GPIO, ADC, PWM, DMA, EXTI, etc. Also choose what type of code you want — Full Project, Driver Library, ISR Handler, etc.
💡 The more specific you select, the better the AI-generated code will be.
04
📡 ADD SENSORS & MODULES
Go to Sensors tab. Browse 65+ industrial sensors. Filter by type or search by name. If your sensor is not listed, use Add Custom Sensor at the bottom.
💡 Select sensor + protocol together for best prompt (e.g. MPU6050 + I2C Master).
05
🔌 SELECT PROTOCOLS
Go to Protocols tab. Select all communication interfaces used — UART, SPI, I2C, CAN, MQTT, RS-485, etc. You can also add a custom protocol not in the list.
💡 Select exactly what your hardware uses. Each selected protocol adds detailed config instructions to the prompt.
06
📊 USE CALCULATORS (Optional)
Use the ADC, Timer, PWM, UART tabs to calculate register values manually. Enter your clock and target frequency — it gives exact PSC, ARR, CCR, BRR values with error percentage.
💡 These calculators work independently. You can use them even without building a prompt.
07
🛡 RTOS & SAFETY (Optional)
Go to RTOS tab. Enable FreeRTOS / Zephyr, add tasks with name, priority and stack size. Also select safety features like Watchdog, Hard Fault Handler, CRC Verification.
💡 For industrial projects always enable IWDG Watchdog and Hard Fault Handler.
08
📝 ADD REQUIREMENTS
Go to Requirements tab. Write your specific requirements — e.g. "Read ACS712 every 50ms, alert if current >5A, send data via MQTT to ThingsBoard". The more detail you add, the better the AI output.
💡 Include timing requirements, threshold values, and output format for best results.
09
🚀 GENERATE & USE PROMPT
Click VIEW or COPY PROMPT at the bottom. In the VIEW modal, click any AI platform (ChatGPT, Claude, Gemini etc.) — prompt is copied automatically. Paste it in the AI chat and get complete firmware code.
💡 For best results use ChatGPT GPT-4o or Claude for STM32/ESP32 firmware code generation.
